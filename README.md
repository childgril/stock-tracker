# 📊 股票損益追蹤系統

一個純前端的股票投資管理網站，可放在 GitHub Pages 上免費使用。

## ✨ 功能

- **多帳戶管理**：可建立多個證券帳戶（元大、國泰、新光等），個別管理損益
- **三種表格匯入**：未實現損益、投資明細、已實現損益
- **自動補利息**：已實現損益自動從投資明細比對出融資利息與融券手續費，補上對應欄位
- **手動調整金額**：每筆已實現損益可手動填入「調整金額」與「備註」（用於當沖融券未買回、券商算錯等情形），重新匯入後會自動還原
- **跨帳戶總覽**：所有帳戶的市值、損益、利息成本一目了然
- **資金趨勢圖**：每次匯入未實現損益會記錄快照，可看資金歷史變化
- **本地儲存 + 備份還原**：資料存在瀏覽器，可匯出 JSON 備份、匯入還原

## 🚀 部署到 GitHub Pages

### 步驟 1：建立 Repository

1. 到 GitHub 建立新的 repository（例如 `stock-tracker`）
2. 把這幾個檔案上傳到 repository 根目錄：
   - `index.html`
   - `style.css`
   - `parsers.js`
   - `app.js`

### 步驟 2：啟用 GitHub Pages

1. 進入 repository → **Settings** → **Pages**
2. **Source** 選 `Deploy from a branch`
3. **Branch** 選 `main`（或 `master`），路徑選 `/ (root)`
4. 儲存後，等 1-2 分鐘
5. 上方會出現你的網址，例如：`https://你的帳號.github.io/stock-tracker/`

## 📥 使用方式

1. 開啟網站後，預設會有一個「主帳戶」，可直接用，或點右上角「＋ 新增」建更多帳戶
2. 切到「⬆️ 匯入資料」頁籤，選擇要匯入的檔案類型：
   - **未實現損益**：當天帳上餘額（會建立快照）
   - **投資明細**：所有交易紀錄（含融資融券利息資訊）
   - **已實現損益**：已賣出的損益紀錄（會自動比對投資明細補上利息）
3. 切換帳戶從上方下拉選單
4. 「💰 已實現損益」頁籤可直接編輯每一筆的「調整金額」與「備註」，會即時儲存

## 🔒 資料儲存

- 所有資料存在**瀏覽器 localStorage**，不會上傳到任何伺服器
- 換瀏覽器、清除快取會遺失資料 → 請定期用「📥 匯出備份」下載 JSON 備份
- 要在另一台電腦或瀏覽器使用，用「📤 匯入還原」載入備份檔即可

## 🏦 支援的券商格式

| 券商 | 未實現 | 投資明細 | 已實現 | 備註 |
|---|---|---|---|---|
| 元大 | ✅ | ✅ | ✅ | 從投資明細比對補上利息 |
| 國泰 | ✅ | ✅ | ✅ | 從投資明細比對補上利息 |
| 新光 | ✅ | ✅ | ✅ | 已實現本身就有利息欄位，直接讀取 |

匯入時系統會**自動辨識**檔案來自哪家券商，不需手動選擇。

新光的檔案使用**民國年**日期格式（例如 `113/12/03`），系統會自動轉成西元年（`2024/12/03`）。

## 🛠 技術

- 純 HTML + CSS + JavaScript，無後端
- [SheetJS](https://github.com/SheetJS/sheetjs) 解析 Excel/CSV
- [Chart.js](https://www.chartjs.org/) 繪製圖表
- 透過 CDN 載入，不需要建構流程

## 📝 已實現損益對應邏輯

比對鍵：**股票代號 + 賣出日期 + 數量**

範例：
- 已實現損益某列：華城 2024/06/18 賣出 1000 股
- 從投資明細找到：華城 2024/06/18 賣出融資 1000 股
- 把該交易的「融資券利息」欄（3,260）填入已實現損益的「利息」欄

**實際盈虧 = 原始盈虧 + 調整金額**

利息與手續費**不會再次扣除**（券商給的盈虧已經扣過），只是顯示讓你知道成本結構。
